  该服务主要采用 express + socket.io-client 
  变量命名规则：驼峰命名
  常量命名：所有字母采用大写，单词与单词之间采用下划线链接
  

  关于配置网络(CPE)的流程
    ·超级管理员录入设备的时候，就已经初始化好设备的端口信息 1个Wan 剩余的都是Lan 名称从GE0开始递增
    ·在CPE上，用户填写好链接类型及相关参数后，进行上网(执行命令 及 重启命令)，客户端刷新页面，获取上网情况。
      ·如果上网不成功，则重新填写
      ·如果上网成功，则进行下一步
    ·上网成功后，进行socket配置，填写密码及远端地址，并且刷新页面，获取链接状态
      ·链接不成功，则重新填写链接信息
      ·链接成功，初始化工作完成，修改配置文件中的CONNECTED，并且同步该设备的联网信息到center
    (如果配置完上网信息，关掉页面，重新登录，需要怎么处理)

  关于配置网络(中心)的流程
    ·CPE初始化完成后，中心在该设备上可以看到上网的配置信息
    ·对该设备的网络端口进行修改
      ·当仅只有一个WAN口时 不让修改
    ·对该上网信息进行修改
      ·修改信息后，发送给CPE
        ·CPE接到指令后，对原有文件进行备份，生成新的文件，并且重启，并且设置定时器
        ·当定时器触发后，判断当前的网络状态
          ·如果成功，则删除备份文件
          ·如果不成功，则删除文件，保留备份文件，并重启，把原有配置再次推送给中心
  (当断网的情况下，node服务不会重启，但socket会断，当重新联网后，socket会重新连接)
